## Four pillars

| Pillar | Meaning |
|--------|---------|
| Encapsulation | Hide data, expose via methods |
| Inheritance | Reuse parent class behavior |
| Polymorphism | Same method, different behavior at runtime |
| Abstraction | Show essentials, hide complexity |
